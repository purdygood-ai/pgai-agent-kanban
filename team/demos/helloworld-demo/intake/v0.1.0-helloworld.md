## Overview

Ship the smallest complete Python program the framework can build end-to-end:
a `hello.py` that prints exactly `Hello, World!` and a unit test that asserts it.
Nothing else. This is the minimal release-workflow requirements document — one
goal, one deliverable, one test. It exists as the copyable smallest-valid-requirements-doc
reference.

## Goals
1. A `hello.py` that when run with `python3 hello.py` prints exactly `Hello, World!`
   (with comma, space, and exclamation point — no variation).

## Deliverables
- `hello.py` — a Python script with a `main()` function that prints exactly
  `Hello, World!` and a `if __name__ == "__main__": main()` guard.
- `test_hello.py` — a unit test that captures stdout and asserts it equals
  `Hello, World!\n` (exact string match including trailing newline; no substring
  matches).

## Acceptance criteria
1. `python3 hello.py` outputs exactly `Hello, World!` (verified by
   `python3 hello.py | grep -Fx "Hello, World!"`).
2. `python3 -m pytest test_hello.py -v` exits 0 with the test passing.
3. The test asserts stdout equality to `Hello, World!\n` — not a substring match.
4. `python3 -m py_compile hello.py` compiles clean.

## Suggested Decomposition
Single CODER task: hello.py + test_hello.py together.

## Notes for CODER
Keep it minimal — this is a reference for what MINIMUM looks like. No argparse, no
logging, no config files, no dependencies beyond the standard library.
