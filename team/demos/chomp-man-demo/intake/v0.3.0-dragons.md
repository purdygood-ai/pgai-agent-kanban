## Overview
Add danger. This release introduces DRAGONS — Chomp Man's pursuers (the genre's "ghosts,"
reimagined as Chomp Man's own enemies). Dragons roam the maze, and if a dragon catches Chomp
Man, the player loses a life. With lives now in play, this release also adds lives tracking
and a game-over state.

## Goals
1. Multiple dragons (suggested 3-4) placed in the maze, each visually distinct (different
   colors/identities — give them names/colors of your choosing).
2. Dragons move through the maze autonomously (they navigate corridors and respect walls,
   same collision rules as Chomp Man).
3. Dragon movement behavior: at minimum, dragons move continuously and change direction at
   junctions (simple wandering AI is acceptable; light pursuit behavior is a nice-to-have).
4. Collision between a dragon and Chomp Man costs a life.
5. Lives tracking (start with 3) shown in the HUD; losing all lives is GAME OVER.
6. On losing a life (but not the last), reset Chomp Man (and optionally dragons) to start
   positions and continue.

## Design / Theming
- Dragons are the Chomp Man enemy — theme them as dragons (color them, name them in code).
  They replace the genre's ghosts; lean into the dragon identity visually.
- Dragon AI: it does not need to be sophisticated. Continuous movement with direction choices
  at intersections is enough for this release; smarter pursuit can be tuned later. Keep the
  behavior in a structure that can be extended (v0.4.0 will need dragons to enter a
  "vulnerable/fleeing" state).
- Keep the game readable: dragons and Chomp Man clearly distinguishable.

## Deliverables
- Updated game code:
  - Dragon entities (multiple), rendered distinctly, placed in the maze.
  - Autonomous dragon movement respecting wall collision, with junction direction choices.
  - Dragon↔Chomp Man collision detection costing a life.
  - Lives counter (start 3) in the HUD; life-loss reset; game-over state when lives reach 0.

## Acceptance criteria
1. `python3 -m py_compile` passes for all changed/added .py files.
2. Multiple dragons render in the maze, visually distinct from each other and from Chomp Man
   (manual visual check).
3. Dragons move autonomously and respect walls (verify dragon movement uses the same
   wall-collision logic; manual confirm they don't pass through walls).
4. A dragon reaching Chomp Man's cell costs a life (verify collision→life-loss logic).
5. Lives start at 3, display in the HUD, decrement on collision; reaching 0 triggers GAME
   OVER (verify the lives + game-over logic).
6. Losing a non-final life resets positions and continues play.
7. Dragon state is structured to allow a future "vulnerable" mode (v0.4.0) — e.g. a per-dragon
   state field, not hardcoded behavior.

## Suggested Decomposition
- CODER-1: dragon entities + rendering + placement.
- CODER-2: autonomous dragon movement (wall-respecting, junction direction choice).
- CODER-3: dragon↔Chomp Man collision + lives + life-loss reset + game-over state + HUD lives.
