## Overview
Bootstrap the Chomp Man project: the minimal scaffolding so the repository is a clean,
runnable starting point before any gameplay is built. No game logic yet — just the project
skeleton, a runnable entry-point stub, and the supporting files a Python project needs. This
establishes the structure every later release builds on.

Chomp Man is a maze arcade game in the spirit of the classic dot-eaters, but it is its own
game — its own name, its own theming (the pursuers are DRAGONS, not ghosts), its own control
scheme, and its own visual identity. This release just lays the foundation.

## Goals
1. A runnable entry point: `python3 pgai-chomp-man.py` launches without error and opens a
   window.
2. A clean Python project skeleton (gitignore, README, dependency declaration).
3. Pygame confirmed as the windowing/graphics library and declared as a dependency.
4. A window that opens, shows a title and a "press Q to quit" message, and closes cleanly.

## Deliverables
- `pgai-chomp-man.py` — entry point. Opens a pygame window (e.g. 800x600), titled "Chomp Man",
  draws a placeholder background and centered text ("CHOMP MAN" + "press Q to quit"), runs a
  clean event loop, and exits cleanly on window-close or the Q key.
- `README.md` — project name, one-paragraph description, how to install deps
  (`pip install -r requirements.txt`), and how to run (`python3 pgai-chomp-man.py`). State the
  Python 3.12+ requirement.
- `requirements.txt` — pin pygame.
- `.gitignore` — standard Python ignores (`__pycache__/`, `*.pyc`, venv dirs, OS cruft like
  `.DS_Store`, editor dirs).

## Acceptance criteria
1. `python3 pgai-chomp-man.py` runs without error on Python 3.12+ (window opens; closes
   cleanly on Q or window-close). Headless/CI note: the script must at least import and
   initialize without crashing; gameplay verification of the window is manual.
2. `test -f README.md && test -f requirements.txt && test -f .gitignore && test -f pgai-chomp-man.py`
   — all present.
3. `grep -qi pygame requirements.txt` — pygame declared.
4. `python3 -m py_compile pgai-chomp-man.py` — compiles clean.
5. README states the run command and Python 3.12+ requirement
   (`grep -q "python3 pgai-chomp-man.py" README.md` and `grep -q "3.12" README.md`).
6. `.gitignore` ignores `__pycache__` (`grep -q "__pycache__" .gitignore`).

## Notes for CODER
Keep this minimal — it is scaffolding, not gameplay. Use pygame for the window. Make the
entry point importable without side effects where reasonable (guard the main loop under
`if __name__ == "__main__":`) so later tests can import it. Window size, font, and colors are
your choice but keep it clean and legible. Do NOT add game logic, dots, dragons, or movement
yet — those are later releases.
