## Overview
Turn the tables. This release adds POWER-UPS — special large collectibles that, when eaten,
temporarily make the dragons vulnerable so Chomp Man can eat THEM for bonus points. This is
the genre's "power pellet" mechanic, reimagined for Chomp Man with its own name and theming.

## Goals
1. A small number of power-ups placed at strategic maze locations (suggested: the corners,
   like the genre convention — 4 power-ups).
2. Eating a power-up triggers VULNERABLE mode for the dragons for a limited time.
3. During vulnerable mode: dragons visibly change appearance (e.g. flee-color), and ideally
   change behavior (flee from Chomp Man — at minimum, they look different and are edible).
4. During vulnerable mode, Chomp Man eating a dragon: the dragon is sent back to its start
   (a "den"/spawn), Chomp Man scores bonus points, and play continues.
5. Vulnerable mode ends after a timer; dragons return to normal (with a brief warning/flash as
   it's about to end — nice-to-have).
6. Eating multiple dragons in a single vulnerable window awards escalating bonus points
   (e.g. 200, 400, 800, 1600) — the genre's combo scoring.

## Design / Theming (Chomp Man identity)
- Name the power-up something on-theme (e.g. "Dragon Bane", "Scale Crystal", "Power Ember") —
  your choice; give it a distinct large-collectible look vs the normal pellets.
- Vulnerable dragons should be clearly distinguishable (color shift) so the player knows they
  are edible.
- Reuse the dragon STATE field added in v0.3.0 — set dragons to VULNERABLE on power-up, back
  to NORMAL when the timer expires or after being eaten + respawned.
- Use the named scoring constants from v0.2.0; add the power-up + dragon-eat bonus values as
  named constants too.

## Deliverables
- Updated game code:
  - Power-ups placed in the maze (distinct from normal pellets), eaten like pellets.
  - Vulnerable-mode trigger + timer; dragon state transitions (NORMAL↔VULNERABLE).
  - Dragon appearance change during vulnerable mode.
  - Eat-dragon-while-vulnerable: send dragon to den, award escalating bonus, continue.
  - Vulnerable-mode expiry returns dragons to normal (optional end-of-mode flash).

## Acceptance criteria
1. `python3 -m py_compile` passes for all changed/added .py files.
2. Power-ups render distinctly from normal pellets and are placed on open cells (manual visual
   + verify placement on open grid cells).
3. Eating a power-up sets dragons to VULNERABLE for a timed window (verify the state
   transition + timer in code).
4. Vulnerable dragons are visually distinct (manual check) and edible.
5. Eating a vulnerable dragon sends it to its den and awards a bonus; escalating bonus for
   multiple eats in one window (verify the combo-scoring logic + named constants).
6. Vulnerable mode expires on the timer and dragons return to NORMAL (verify the timer expiry
   path).
7. A dragon in NORMAL state still costs Chomp Man a life on collision (the v0.3.0 behavior is
   preserved when not vulnerable).
8. Scoring additions use named constants (consistent with v0.2.0).
