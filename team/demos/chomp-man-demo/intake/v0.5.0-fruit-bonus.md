## Overview
Add the reward. This release introduces BONUS FRUIT — high-value collectibles that appear
periodically in the maze for a limited time, giving Chomp Man a reason to take risks for big
points. This is the genre's fruit-bonus mechanic (cherries, etc.), with Chomp Man's own
theming and point values.

## Goals
1. Bonus fruit appears periodically at a set maze location (suggested: near the center/den),
   for a limited time, then disappears if not eaten.
2. Eating the fruit awards a high point value (more than pellets or single dragon-eats).
3. Fruit appearance is tied to game progress (suggested: appears after a threshold of pellets
   eaten, and/or on a timer) — your design, but it should feel earned, not constant.
4. Different fruit types with different point values that escalate as the player progresses
   (the genre cycles cherry → strawberry → ... ; theme yours as you like).
5. The fruit is visible and clearly a special, time-limited collectible.

## Design / Theming (Chomp Man identity)
- Theme the fruit to Chomp Man — they can be literal fruit (cherries, etc.) or something on-
  theme (e.g. dragon eggs, gems). Your choice; make them visually special and distinct from
  pellets and power-ups.
- Point values are your design but should be the highest standard collectible values in the
  game and use named constants (consistent with v0.2.0 / v0.4.0). Suggested escalating tiers
  (e.g. 100, 300, 500, ...).
- Timing: fruit should appear for a limited window and vanish if not eaten — a risk/reward
  decision for the player.

## Deliverables
- Updated game code:
  - Fruit spawn logic (periodic / progress-based), at a set location, for a limited time.
  - Fruit rendering (distinct, special-looking) + despawn when its window expires.
  - Eat-fruit detection awarding high points (named constants).
  - Escalating fruit tiers/values as the player progresses.

## Acceptance criteria
1. `python3 -m py_compile` passes for all changed/added .py files.
2. Fruit appears periodically (progress- and/or timer-based) at the intended location and
   despawns after its window if uneaten (verify the spawn/despawn logic + timer).
3. Fruit renders distinctly from pellets and power-ups (manual visual check).
4. Eating fruit awards high points using named constants (verify the scoring + constants).
5. Fruit values escalate across tiers as the player progresses (verify the tier logic).
6. Regression: pellets, power-ups, dragons, lives, vulnerable mode, and board-clear all still
   work (this release adds fruit without breaking prior mechanics).

## Suggested Decomposition
- CODER-1: fruit spawn/despawn logic (timing + location + limited window).
- CODER-2: fruit rendering + eat detection + high-value named-constant scoring.
- CODER-3: escalating fruit tiers/values + regression-safe integration.

constants and the tier progression readable.
