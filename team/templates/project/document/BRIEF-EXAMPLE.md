# Brief example (document workflow)

This is a sample brief showing the format the WRITER expects. Operators
embed brief content INSIDE the requirements file under "## Brief" section
(see templates/REQUIREMENTS-TEMPLATE.md).

For complex briefs that span many pages, you may instead reference an
attached file from the requirements doc:
  ## Brief
  See: artifacts/v0.1.0/brief.md

Sample short-form brief content:

  Audience: technical decision-makers evaluating agent frameworks
  Tone: confident but not hype-laden; technical but accessible
  Length: 18-22 pages (~6000-7500 words)
  
  Outline:
    1. Executive summary (~1 page)
    2. The problem space (~2 pages)
    ...
  
  Source material: ARCHITECTURE.md, ROADMAP.md, real production metrics
  
  Voice: third person ("the framework"); no first-person plural
  
  Avoid: marketing superlatives, competitor name comparisons, invented quotes
