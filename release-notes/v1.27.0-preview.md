pgai-agent-kanban v1.27.0 (preview)

The container-and-credibility era: everything from v1.23.0 to
v1.27.0, built — as always — by the framework itself, on its own
board.

Highlights since v1.23.0:

- Containerized deployment as a first-class target: Debian and
  RHEL9/UBI9 image flavors, a fail-loud entrypoint verifying the
  four canonical mounts, pseudocron as PID 1, and a runbook
  (docs/docker.md) held to the executable-documentation standard.
- Relocatable scheduling: pseudocron configs carry root-relative
  commands and run from the resolved kanban root — the same config
  works on a host, in a container, or anywhere the install moves.
- Coding standards, self-enforced: docs/coding-standards.md is
  binding on every code-producing agent, backed by a lint fleet
  (help-presence, comment-provenance, import-completeness) that the
  framework runs against its own tree — and has already used to
  reject its own work.
- Executable documentation: a doc-lint harness runs every fenced
  command in the public docs verbatim; links, fossils, and feature
  coverage are gated the same way.
- Committed VERSION: releases stamp their identity into the tree at
  ship time; installs and containers report exactly what they run.
- Release-engineering hardening: cancelled version numbers free
  cleanly, bug fixes always precede feature work at selection, and
  the recovery drill is race-guarded — lessons from production,
  encoded as fixtures.
- Operator experience: an operator dashboard window (full board
  plus a working shell — the container operator's workspace),
  configurable status glyphs, force-reset with worktree cleanup,
  and complete in-container test tooling.

Every release between the public tags was decomposed, implemented,
verified, and shipped autonomously by the six-agent chain, with the
ship-policy matrix and the human-approval gate deciding what
crossed the line.

This is a preview build for field evaluation ahead of the formal
v1.27.0 release. The API contract remains ICD 1.2.0 — no changes
since v1.23.0.
